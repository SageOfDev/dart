### 컴파일러
1. 멀티플랫폼으로 알아서 컴파일 해줌
- 거의 모든 곳에서 사용할 수 있음
2.  JIT(just-in-time), AOT(ahead-of-time) 둘 다 적절하게 사용함.
- AOT
  - 실행 전에 컴파일을 함.
  - 컴파일은 시간을 많이 사용한다.
    - UI에선 별로 좋은 방법이 아니다.
    - 바로 바로 볼 수 없기에 개발 중엔 좋지 않음
    - 배포 후에 사용함. 앱이 빨리 돌아가야 하므로.
- JIT    
  - 필요한 부분을 그때 그때 컴파일함.
  - 개발 중에 사용함.
3. null safety
- null 값을 참조하면 프로그램이 고장날 수 있음
- dart를 이를 예방하기 위해 null safety를 사용한다.
